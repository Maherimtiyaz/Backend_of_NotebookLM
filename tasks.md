# Project Completion Tasks

## 1. Document Upload System
### Tasks:
1. Create an API endpoint for uploading documents.
   - **Learning Resource**: [FastAPI File Uploads](https://fastapi.tiangolo.com/tutorial/request-files/)
2. Validate uploaded files (e.g., file type, size).
   - **Learning Resource**: [Pydantic Validation](https://pydantic-docs.helpmanual.io/usage/validators/)
3. Save uploaded documents to the database.
   - **Learning Resource**: [SQLAlchemy ORM Basics](https://docs.sqlalchemy.org/en/14/orm/quickstart.html)

## 2. Document Storage
### Tasks:
1. Design the database schema for storing documents.
   - **Learning Resource**: [Database Design Basics](https://www.postgresql.org/docs/current/datatype.html)
2. Implement the database layer for document storage.
   - **Learning Resource**: [SQLAlchemy Models](https://docs.sqlalchemy.org/en/14/orm/tutorial.html)

## 3. Document Processing Layer
### Tasks:
1. Create a service to chunk documents into smaller parts.
   - **Learning Resource**: [Python Text Processing](https://docs.python.org/3/library/re.html)
2. Integrate the chunking service with the document upload system.
   - **Learning Resource**: [FastAPI Dependency Injection](https://fastapi.tiangolo.com/tutorial/dependencies/)

## 4. Query System for User Documents
### Tasks:
1. Design the query system architecture.
   - **Learning Resource**: [System Design Basics](https://www.educative.io/courses/grokking-the-system-design-interview)
2. Implement the query system for retrieving and searching user documents.
   - **Learning Resource**: [SQLAlchemy Querying](https://docs.sqlalchemy.org/en/14/orm/query.html)
3. Add AI-based querying support (future scope).
   - **Learning Resource**: [OpenAI API](https://platform.openai.com/docs/)

## 5. Testing and Deployment
### Tasks:
1. Write unit tests for all APIs and services.
   - **Learning Resource**: [Pytest Documentation](https://docs.pytest.org/en/7.0.x/)
2. Set up CI/CD pipeline for automated testing and deployment.
   - **Learning Resource**: [GitHub Actions](https://docs.github.com/en/actions)
3. Deploy the application to a cloud platform (e.g., AWS, Azure, Heroku).
   - **Learning Resource**: [FastAPI Deployment](https://fastapi.tiangolo.com/deployment/)
